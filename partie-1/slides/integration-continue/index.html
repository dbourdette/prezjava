Intégration continue

Une pratique agile

L'intégration continue est une pratique agile qui vise à intégrer régulièrement tout le travail d'une équipe.

La fréquence d'intégration est variable, allant d'une fois par jour à de multiples fois par jour.

Le but est de détecter l'apparition d'anomalies le plus tôt possible.

La définiton d'intégration couvre généralement la récupération du code source, l'exécution des tests et le build du produit.

If it hurts, do it more often (Kent Beck)

http://martinfowler.com/articles/continuousIntegration.html

Bénéfices

Les bénéfices de l'intégration continue sont multiples :

  • Pas de surprises en fin de projets
  • Facilite la tenue d'un planning
  • Réduction du risque
  • Permet de déployer en production plus souvent

Il faut noter que la force de l'intégration continue repose sur la qualité des tests qui accompagne le code source, car se sont qui sont garant de la qualité du code.

http://martinfowler.com/articles/continuousIntegration.html

Pratiques clés

Voici les pratiques associées à l'intégration continue:

  • Un seul repository pour le code source
  • Build automatisé
  • Build exécutant les tests
  • Tout le monde commit quotidiennement dans la branche maitre
  • Chaque commit dans la branche maitre déclenche un build
  • Le build doit être rapide
  • Les tests doivent être exécuté dans un environnement proche de la configuration de production
  • Rendre disponibles facilement les artefacts
  • Tout le monde peut voir le résultat des builds
  • Le déploiement doit être automatisé

Les outils de build

Qualités d'un outils de build

Un outils de build est du code dont le but est de construire le livrable (jar, war, tar.gz, zip...).

Il doit pouvoir s'exécuter en une seule ligne de commande et produire le livrable qui un unique fichier.

Ainsi, le build est facilement automatisable et le résultat peut être numéroté (ex : build-1.1.3.zip) et archivé.

Idéalement, il exécute les jeux de tests et produit divers rapports qui sont exploités pour le suivi de qualité ou la documentation.

Principaux outils pour java

Voici les principaux outils de build qui peuvent être utiliser pour les projets java.

ant

le premier outils de build en java pour java. Il utilise un style impératif comme make. Le script de build est une liste de commandes en xml.

maven

un outils de build qui utilise un style déclaratif. Sa force réside dans sa gestion des dépendances et son intégration dans la chaîne de montage logicielle.

ivy

pas vraiment un outils de build mais plutôt un gestionnaire de dépendances.

gradle

assez proche de maven car c'est une approche déclarative. Plus moderne, il repose sur un DSL et est plus flexible. Il est cependance moins répendu et donc moins supporté pas les logiciels tiers.

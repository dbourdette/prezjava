Le langage java

Le langage Java

Le Java est un langage orienté objet :

  • encapsulation
  • polymorphisme
  • héritage
  • appels dynamiques

Java n'est pas un pur language objet car il inclut des types de base (int, long...) et on ne peut pas surcharger certains opérateurs (+, -...).

C'est un language fortement typé : les contrôles sont effecutés à la compilation.

Le code source est compilé en byte code ce qui le rend portable sur tous les environnements disposant d'une machine virtuelle Java.

La version courante est 1.6 et la version 1.7 est en cours de spécification.

http://en.wikipedia.org/wiki/Object-oriented_programming

Compilation et méthode main

Java est un langage compilé. Mais contrairement au C qui est compilé en instructions natives, Java est compilé en bytecode.

Cela lui permet d'être portable et exécutable sur toutes les machines où tourne une machine virtuelle java (jvm).

La commande javac compile les fichiers .java en .class qui sont alors exécutables par la commande java.

Afin de pouvoir être exécuté, un programme java doit posséder une méthode main.

public static void main(String[] args) { ... }

Le chargement du code java est dynamique : il est chargé au fur et à mesure du besoin.

La machine virtuelle regarde dans le classpath afin de charger du code. Si une classe demandée n'est pas trouvée alors une erreur est remontée.

java.lang.ClassNotFoundException: path.to.my.unknown.Code

Package, class et import

Les classes sont rangés dans des packages correspondants à l'arborescence des répertoires.

package edu.fr.test

public class Test {
   ...
}

Si on souhaite utiliser une classe depuis une autre, il suffit de faire un import de la classe

import edu.fr.test.Test

Ou même de tout le package

import edu.fr.test.*
classe Object

Toutes les classes héritent de la classe Object qui définit les méthodes suivantes :

  • getClass : la classe de l'object
  • toString : une représentation en chaîne de caractères de l'objet
  • equals : test d'égalité
  • hashCode : une valeur de hash (un entier qui représente l'état de l'objet)
  • clone : retourne un copie
  • wait / notify : permet de bloquer ou débloquer le thread courant
  • finalize : appelé quand l'objet va être supprimé de la mémoire

En pratique, on surcharge fréquement equals, hashCode (qui vont de pair) et toString.

interface

Une interface définit un ensemble de méthodes et de constantes

public interface MailService {
   public static final int SOME_NAME = 10;

   public void send(Mail mail);

   public List<Mail> retrieve();
}

Il n'est pas possible d'instancier une interface.

Une classe peut implémenter une interface : elle doit alors implémenter toutes les méthodes.

enum

Un enum est une classe qui ne possède qu'un nombre prédéfini d'instance

public enum Gender {
	MALE, FEMALE, UNKNOWN;
}

Les instances MALE, FEMALE et UNKNOWN n'existe qu'une seule fois dans toute la jvm.

On peut donc faire des tests d'égalité par référence.

Gender gender1 = Gender.MALE;
...
Gender gender2 = Gender.MALE;

gender1 == gender2;
Annotations

Les annotations permettent de marquer des classes ou des méthodes afin de les distinguer.

Du code peut alors retrouver ces entités marquées et appliquer des traitements.

Une annotation se définit comme suit.

public @interface LookAtMe {
	String why();

	String comments() default "none";
}

Et s'utilise simplement.

@LookAtMe(why = "why not")
    public void save() {
   ...
}
Les stacktraces

Quand une erreur survient, une exception est levée.

Une exception est un type particulier d'objet instancié en cas de problème et qui peut être lancé via la synthaxe :

throw new IllegalArgumentException("param can not be null");

La méthode printStackTrace d'une exception permet d'obtenir la stacktrace :

java.io.FileNotFoundException: test.txt
   at java.io.FileInputStream.(FileInputStream.java)
   at java.io.FileInputStream.(FileInputStream.java)
   at Test.readFile(Test.java:19)
   at Test.main(Test.java:7)

Une stactrace contient le nom de l'erreur, un message et l'état complet de la pile d'appel au moment ou l'exception a été lancée.

C'est donc un outils très complet afin de comprendre exactement ce qui a déclanché l'erreur.

La gestion de la mémoire

Une application java réserve la mémoire en masse et la redistribue ensuite en fonction du besoin.

Il n'y a donc pas besoin d'écrire du code afin de réserver ou libérer de la mémoire.

La mémoire réservée est découpée en plusieurs zones, selon l'anciennté des objets.

La garbage collector a pour charge de régulièrement regarder les objets devenus inutiles et de libérer la mémoire correspondante.

java et jee

En plus du langage, java propose une suite logicielle pour entreprise : Java Enterprise Edition (jee).

C'est une galaxie de service extrêment vaste donc en voici quelques élements :

  • Conteneurs de composants : servlets, portlets et ejb
  • JPA : persistance object - relationnel
  • JAX-WS : les services SOAP
  • JAX-RX : les servcies REST
  • JAXB : binding java - xml
  • JDBC : java database connectivity
  • JMS : java messaging service
  • JSF : java server faces
  • Bean validation : annotations pour la validation
  • javaMail : envoi et réception de mail
  • JMX : standardisation du management et supervision des applications
  • JNDI : java naming and directory interface
  • Technologies de rendu : fichiers jsp, custom tags et expression language
Environnements managés

Lorsqu'un application est déployée dans un serveur jee, elle ne possède pas de méthode main().

C'est le serveur qui charge dynamiquement les objets et les appelle en fonction du besoin.

Dans ce cadre, le serveur est appelé conteneur et les objest composants.

Un conteneur est un environnement qui manage des composants.

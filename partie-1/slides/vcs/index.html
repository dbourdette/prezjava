Gestion de sources

Qualités d'un outils de gestion de sources

Un VCS présente des qualités similaires à une base de données (consistance, durabilité, transactions...)

C'est le coffre fort d'une équipe de développement. C'est la que tout doit être rangé afin que l'on puisse tout y retrouver et dans n'importe quelle version.

Il doit être simple d'interagir avec et simple de restaurer un état passé.

Tous types de sources peuvent être stoqués dans un VCS : code, fichiers de configuration, scritps utilitaires...

Principaux VCS

CVS

Le gestionnaire de sources gratuit "historique". Il est aujourd'hui très dépassé par des solutions comme subversion ou git.

Subversion

Il reprend les mêmes principes que CVS mais en corrige les principaux défauts.

Il propose des conventions simples et faciles à manipuler.

Si il est installé derrière un apache 2, il est possible de naviguer dans les sources avec un browser.

Git

Git est un DVCS et suit donc une philosophie différente.

Il multiplie les capacités de travail dans des branches (locales ou distantes) permettant de travailler simultanément sur plusiseurs sujets en parallèle.

Il est particulièrement adapté à du travail sur des projets open source où les bureaux sont dématérialisés et les collaborateurs multiples.

Github

Un versioning control system (VCS) présente des qualités similaires à une base de données (consistance, durabilité, transactions...)

C'est le coffre fort d'une équipe de développement. C'est la que tout doit être rangé afin que l'on puisse y retrouver tout ce qu'on a écrit.

Il doit être simple d'interagir avec et simple de restaurer un état passé.

Tous types de sources peuvent être stoqués dans un VCS : code, fichiers de configuration, scritps utilitaires...

Les principaux gestionnaires de sources libres sont les suivants :

  • CVS : Le gestionnaire de sources gratuit "historique". Il est aujourd'hui très dépassé par des solutions comme subversion ou git.
  • Subversion : Il reprend les mêmes principes que CVS mais en corrige les défauts. Il propose des conventions simples et faciles à manipuler. Si il est installé derrière un serveur http apache 2, il est possible de naviguer dans les sources avec un browser.
  • Git : Git est un distributed VCS (DVCS) et suit donc une philosophie différente. Il multiplie les capacités de travail dans des branches (locales ou distantes) permettant de travailler simultanément sur plusiseurs sujets en parallèle. Il est particulièrement adapté à du travail sur des projets open source où les bureaux sont dématérialisés et les collaborateurs multiples.

Compte github

Github est un site web qui propose une interface web communautaire par dessus git. Github est gratuit pour les projets open source et simple à utiliser. Cela en fait un outils de référence pour les projets open source.

C'est l'outils que nous allons utiliser pour notre projet.

Créez un compte github et configurez une clée ssh en suivant le tutorial github

Création du repository todooz sur github

Depuis le dashboard, créez un repository todooz.

On doit donc obtenir un repository de la forme git@github.com:{user}/todooz.git

On peut choisir un autre nom pour le repository. Dans ce cas, il faut adapter les références à github pour toutes les manipulations à venir.

git init

Afin de pouvoir pusher notre projet vers github, il faut d'abord initialiser notre répertoire projet.

todooz $ git init
Initialized empty Git repository in /path/to/projects/todooz/.git/

.gitignore

Certains fichiers ne doivent pas être envoyé vers le VCS : les fichiers générés lors de la compilation ou bien les fichiers générés par maven ne font pas partie des sources.

On crée le fichier .gitignore à la racine du projet afin de les exclure.

# maven
target/*

# intellij
.idea
.iml

# eclipse
.project
.classpath

Le répertoire target contient le résultat de la compilation et autres fichiers générés lors du build.

.idea et .iml sont la définition du module sous intellij.

.project et .classpath sont la définition du projet eclipse et sont générés par maven (mvn eclipse:eclipse).

Il faudra ajouter d'éventuels autres patterns si le besoin se présentait.

git add

La commande git status doit donc afficher à peu près cela.

todooz $ git status
# On branch master
#
# Initial commit
#
# Untracked files:
#   (use "git add <file>..." to include in what will be committed)
#
#       .gitignore
#       pom.xml
#       src/
nothing added to commit but untracked files present (use "git add" to track)

On ajoute donc tout nos fichiers et on commit.

toddoz $ git add *
toddoz $ git add .gitignore
toddoz $ git commit -m "initial commit"

git remote add

Il nous reste donc a relier notre master local au origin/master

toddoz $ git remote add origin git@github.com:{user}/todooz.git

git push

Afin d'envoyer les fichiers vers github, on utilise la commande push.

toddoz $ git push origin master

Les fichiers sont maintenant sur le repository github.

Afin de simplifier les push et les pull, on peut ajouter un lien par défaut entre le master local et le master sur origin.

toddoz $ git config branch.master.merge refs/heads/master

Maintenant git push et pit pull feront automatiqment le lien avec origin. La commande de push devient donc simplement :

toddoz $ git push

A partir de maintenant, il est conseillé de commiter aussi souvent que nécessaire.

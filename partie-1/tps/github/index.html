Création du repository blog sur github

Depuis le dashboard, créer un repository blog.

On doit donc obtenir un repository de la forme git@github.com:{user}/blog.git

git init

Afin de pouvoir pusher notre projet vers github, il faut d'abord initializer notre répertoire projet

blog > git init
Initialized empty Git repository in /path/to/projects/blog/.git/

.gitignore

Afin d'être sûr de ne pas commiter les mauvais fichiers, on crée le fichier .gitignore

.project
.classpath
target/*

git add

La command git status doit donc afficher à peu près cela.

blog > git status
# On branch master
#
# Initial commit
#
# Untracked files:
#   (use "git add ..." to include in what will be committed)
#
#       .gitignore
#       pom.xml
#       src/
nothing added to commit but untracked files present (use "git add" to track)

On ajoute donc tout nos fichiers et on commit

blog > git add *
blog > git add .gitignore
blog > git commit -m "initial commit"

git remote add

Il nous reste donc a relier notre master local au origin/master

blog > git remote add git@github.com:{user}/blog.git
blog > git push origin master

Les fichiers sont maintenant sur le repository github.

Afin de simplifier les push et les pull, on peut ajouter le lien entre le master local et le master sur origin.

blog > git config branch.master.merge refs/heads/master

Maintenant git push et pit pull depuis la branche locale master travaillerons avec la branche master distante.

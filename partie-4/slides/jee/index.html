JEE

The one stop shop

Les spécifications jee sont censées couvrir tous les besoins "entreprise".

  • EJBs : les composants
  • JPA : la persistance en base de données relationnelles
  • JAX-WS : les services SOAP
  • JAX-RS : les services REST
  • JAXB : binding java - xml
  • JDBC : Java Database Connectivity
  • JMS : Java Messaging Service
  • JSP : Java Server Pages
  • JSF : Java Server Faces (applications web à composants)
  • Bean validation : annotations pour la validation
  • JNDI : Java Naming and Directory Interface
  • javaMail : envoi et réception de mail
  • JMX : standardisation du management des applications
Les EJBs

Les enterprise java beans sont au coeur des serveurs d'application : ils composent les couches métier de l'application.

La version 3.1 (jee 6) facilite énormément l'écriture d'EJBs.

Une conteneur d'EJBs propose de nombreux services :

  • Cycle de vie : gestion d'un cycle de vie pour les composants
  • Injection de dépendances : aide à la constuction de l'application
  • Intercepteurs : factorisation du code transverse
  • Pooling : pool d'objets afin de gérer la concurrence
  • Transactions : prise en main des transactions
  • Appels asynchrones : possiblité de déclencher des traitements asynchrones
  • Sécurité : utilisateurs et rôles
  • Communication distante : facilitation des appels à des services distants
  • Messaging : envoi et consommation de messages
Les types d'EJBs
  • Entity beans (@Entity) : les entités pour la persistance de données (lié a JPA)
  • Session beans : les services
    • @Singleton : les services sans état
    • @Stateless : les services sans état (disponibles dans un pool)
    • @Stateful : les services avec état conversationel (gestion de session)
  • Message driven beans (@MessageDriven) : consommateurs de messages asynchrones

Les EJBs ont longtemps été une spécification lourde à respecter.

Avec la version 3.1, ils ressemblent beaucoup à des composants spring.

Cependant, il faut un conteneur d'application pour les EJBs alors qu'une simple jvm suffit à spring.

Packaging

Avant les EJBs 3.1, il était obligatoirement compliqué.

Avec la spécification 3.1, pour une application web, un war peut tout contenir.

C'est une simplification majeure dans l'histoire des EJBs.
